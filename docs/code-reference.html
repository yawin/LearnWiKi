6. 关键业务流程

6.1 学习闭环

目标创建 ──► AI 自动关联知识点 ──► 用户阅读 (已读切换) │ │ │ ▼ │ review_schedule 创建 │ │ │ ▼ │ 知识点选择题复习 (ReviewQuiz) │ │ │ ▼ │ submit_review_feedback │ (SM-2 计算下次复习时间) │ │ │ ▼ │ 所有知识点已读+已复习 │ │ └───────────────────────────────────┘ │ ▼ 考试解锁 (创建试卷) │ ▼ 考试答题 → 成绩报告 │ ▼ 掌握度标记 → 循环复习

6.2 内容捕获 → 知识编译

剪贴板/截图 ──► 弹出确认窗口 ──► 保存到 captured_content │ ▼ AI 编译 (wiki_engine.rs) │ ┌────────┼────────┐ ▼ ▼ ▼ 评估 发现关联 创建页面 (是否需要 (与现有知识 (实体/概念/ 新页面) 建立链接) 主题/对比) │ ▼ wiki_edges 图谱

6.3 复习题型轮转算法 (SM-2 + Format Rotation)

ALL_FORMATS = ["choice", "judgment", "essay"] 第 1 次复习 (review_count==0): → 随机选择 "choice" 或 "judgment" 第 2~3 次 (review_count<=2): → 从未使用的题型中随机选择 → 排除 "essay" (需要基础知识) 第 4+ 次 (review_count>=3): → 排除最近 2 次使用的题型 → "essay" 可用 (已具备基础) 质量反馈 → SM-2 算法: quality=0: 间隔重置为 1 天 quality>=1: 间隔×难度系数 (interval × ease_factor) ease_factor: 答对 +0.1, 答错 -0.2, 最低 1.3

7. AI 提示词组织

7.1 提示词架构

所有提示词集中在两个文件，按场景划分。每个函数接受 locale 参数实现中英文双语。

ai/prompts.rs ──── 通用提示词 ├── weekly_report_system_prompt() 周报生成 (系统级) ├── content_summarize_prompt() 内容摘要 ├── topic_extraction_prompt() 话题提取 └── topic_clustering_prompt() 话题聚类 ai/wiki_prompts.rs ──── Wiki 提示词 ├── assessment_system_prompt() 评估：是否需要为新内容创建 Wiki 页面 ├── compile_discover_system_prompt() 发现：内容与现有知识的联系 ├── compile_execute_system_prompt() 执行：生成页面内容 (标题/摘要/正文) ├── query_retrieve_system_prompt() 检索：从知识库检索相关页面 ├── query_answer_system_prompt() 回答：基于检索结果回答问题 ├── query_rewrite_system_prompt() 改写：将自然语言问题改写为搜索引擎查询 └── lint_system_prompt() 纠错：检测知识库结构问题 ai/client.rs ──── AI 客户端路由 └── call(system_prompt, user_message) → Provider 路由 ├── Anthropic → call_anthropic() ├── OpenAI → call_openai() ├── Gemini → call_gemini() └── OpenRouter → call_openrouter() commands/review.rs ──── 复习题目生成 (内联提示词) ├── generate_quiz_questions() 选择题出题 ├── generate_ordering_steps() 排序题出题 ├── generate_error_hunt() 找茬题出题 ├── generate_cloze() 填空题出题 └── generate_explain_review() 解释题出题

7.2 提示词示例

Wiki 编译 - 评估 (assessment_system_prompt)

你是一个知识管理助手。分析以下内容片段，判断是否需要为其创建独立的 Wiki 页面。

评估标准：
1. **实体/概念**：是独立的知识单元（人物、工具、术语、理论）
2. **可复用性**：在未来可能被其他内容引用
3. **信息密度**：包含足够的细节值得单独记录

返回 JSON：
{
  "needs_page": true/false,
  "confidence": 0.0-1.0,
  "title": "建议的页面标题",
  "page_type": "concept|entity|source|comparison|overview",
  "reasoning": "简短理由"
}

Wiki 编译 - 执行 (compile_execute_system_prompt)

你是一个知识百科编纂者。为以下主题创建结构化的 Wiki 条目。

页面标题：{title}
相关材料：{related_contents}
相关联的页面：{linked_pages}

请生成：
- 摘要 (50-100 字)
- 概念阐述 (200-400 字)
- 详细展开 (300-600 字)
- 延伸阅读建议

返回 JSON: { "summary": "...", "concept": "...", "detail": "...", "extend": "..." }

选择题出题 (generate_quiz_questions)

You are a quiz generator. Generate N multiple-choice question(s)
with 4 options each, based on the knowledge content provided.
Each question must have exactly one correct answer.

Return a JSON array:
[{
  "stem": "question text",
  "options": ["A", "B", "C", "D"],
  "correct_index": 0,
  "explanation": "why this answer is correct"
}]

周报生成 (weekly_report_system_prompt)

你是一个个人知识助理。基于用户本周捕获的内容，生成周报。

要求：
- 总结本周的主题趋势
- 标记重要的发现和洞察
- 识别可能被忽视的内容
- 用简洁、亲切的语气

返回 JSON: { "summary": "...", "highlights": [...], ... }

7.3 提示词设计原则

原则说明
系统/用户分离system_prompt 定义角色和规则，user_message 传入具体数据
JSON 输出约束几乎所有提示词要求 JSON 格式输出，便于程序解析
双语支持通过 locale 参数切换中英文角色描述和输出要求
结构化角色每个场景有明确的"人格"定义 (编纂者/出题助手/知识分析师)
评分标准明确如评估提示词列出 confidence 0-1、page_type 枚举等具体约束
内容分离提示词本身不包含具体数据，通过参数注入 wiki 内容/用户数据
